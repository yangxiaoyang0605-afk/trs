# 后台管理分支页面设计说明（嵌入现有投诉工单系统）

## 1. 页面定位

本后台管理不是独立系统，而是嵌入现有“智慧消保平台 / 投诉工单系统”中的一个功能分支。

```text
智慧消保平台
├── 投诉工单
│   ├── 工单录入
│   ├── 工单处理
│   └── 投诉台账
├── 数据分析
│   ├── 业务报表
│   └── 统计看板
└── 后台管理
    ├── 用户管理
    ├── 角色权限管理
    ├── 机构管理
    └── 短信模板维护
```

设计要求：

- 不新建独立后台系统。
- 保留现有系统左侧导航、顶部栏、消息提醒、个人信息等统一框架。
- 仅在主内容区切换后台管理页面。
- 后台管理入口仅对“超级管理员”可见。
- 页面风格严格沿用现有日照银行智慧消保平台 UI。

---

## 2. 核心权限模型

系统权限拆成 4 层：

```text
用户
  ↓ 绑定
角色
  ↓ 配置
菜单权限 + 按钮权限 + 数据范围 + 业务范围
```

### 2.1 菜单权限

控制用户是否能看到某个菜单，例如：

- 工单录入
- 我的待办
- 本机构/下辖工单
- 台账查询
- 报表查询
- 统计看板
- 用户管理
- 角色权限管理
- 机构管理
- 短信模板维护

### 2.2 按钮权限

控制用户进入页面后可以执行什么操作，例如：

- 新增
- 编辑
- 查看
- 派发
- 转办
- 退回
- 审核
- 关单
- 回访
- 导出
- 启用
- 停用
- 分配角色

### 2.3 数据范围

建议统一为：

```text
仅本人
本机构
本机构及下辖机构
指定机构
全行
```

### 2.4 业务范围

控制用户在业务中承担什么职责。

## 3. 角色定义

### 3.1 机构管理员

定位：负责机构投诉统筹、分派和管理。

主要能力：

- 单笔/批量录入工单
- 查看本人待办
- 查看本机构及下辖机构工单
- 退回
- 转派
- 查看工单详情
- 关单审核
- 台账查询
- 台账导出
- 报表查询
- 统计看板
- 待办/超期提醒

默认数据范围：

```text
本机构 + 下辖机构
```

---

### 3.2 投诉处理员

定位：负责具体投诉工单处理。

主要能力：

- 查看本人待办
- 查看本机构/授权范围内工单
- 查看工单详情
- 填写处理结果
- 联系客户
- 上传处理材料
- 转办
- 提交审核
- 回访登记
- 待办/超期提醒

默认不具备：

- 单笔/批量录入
- 关单审核
- 台账导出
- 后台管理

默认数据范围：

```text
仅本人
或
本机构授权范围
```

---

### 3.3 超级管理员

定位：系统最高权限角色，只负责系统配置和维护，不直接参与具体投诉办理。

主要能力：

- 用户管理
- 角色权限管理
- 机构管理
- 短信模板维护
- 查看全局配置

建议规则：

```text
默认不参与工单业务处理
默认不作为工单处理人
默认不进入业务待办
```

## 5. 角色权限管理页面

这是后台管理中最重要的页面。

### 5.1 页面布局

采用“左侧角色列表 + 右侧权限详情”的双栏布局。

```text
┌─────────────────────────────────────────────┐
│ 后台管理 / 角色权限管理                      │
│ 角色权限管理                                 │
├───────────────┬─────────────────────────────┤
│ 角色列表      │ 角色详情                      │
│               │                               │
│ 机构管理员    │ 基本信息                      │
│ 投诉处理员    │ 菜单权限                      │
│ 超级管理员    │ 按钮权限                      │
│               │ 数据范围                      │
│ [+ 新增角色]  │ 关联用户                      │
└───────────────┴─────────────────────────────┘
```

### 5.2 左侧角色列表

每张角色卡片显示：

```text
机构管理员
18人
本机构及下辖机构
启用
```

```text
投诉处理员
280人
按机构授权
启用
```

```text
超级管理员
2人
全行系统配置
启用
```

支持：

- 点击切换角色
- 搜索角色
- 新增角色
- 编辑角色
- 启用/停用

不建议使用复杂表格，左侧卡片更适合现有系统风格。

### 5.3 右侧角色详情

顶部展示：

```text
角色名称：机构管理员
角色编码：ORG_ADMIN
角色状态：启用
关联用户：18人
```

按钮：

```text
[编辑基本信息] [复制角色] [停用角色]
```

下方使用 4 个 Tab：

```text
菜单权限 | 按钮权限 | 数据范围 | 关联用户
```

要求：

- 支持“全选 / 清空”

- 修改后点击保存生效

## 7. 按钮权限设计

按业务模块分组。

### 工单录入

```text
☑ 查看
☑ 单笔录入
☑ 批量导入
☑ 保存草稿
☑ 派发
```

### 工单处理

```text
☑ 查看详情
☑ 填写处理结果
☑ 外呼
☑ 发送短信
☑ 上传沟通记录
☑ 转办
☑ 提交审核
☐ 关单审核
```

### 台账

```text
☑ 查询
☑ 查看详情
☐ 导出
☑ 回访
```

原则：

- 页面可见不等于按钮可操作。
- 按钮权限必须单独配置。
- 高风险动作单独控制，例如导出、关单审核、角色分配。

## 8. 数据范围配置

采用单选 + 条件配置。

```text
数据范围

○ 仅本人
○ 本机构
● 本机构及下辖机构
○ 指定机构
○ 全行
```

选择“指定机构”后显示：

```text
[选择机构]

已选择：
东港支行
莒县支行
五莲支行
```

规则：

- 数据范围决定工单列表、台账、统计看板的可见数据。
- 机构管理员默认：本机构及下辖机构。
- 投诉处理员默认：仅本人或本机构授权。
- 超级管理员：全行系统配置，但默认不进入业务处理。

## 9. 关联用户设计

Tab 名称：

```text
关联用户（18）
```

列表字段：

| 姓名 | 工号 | 所属机构 | 当前角色 | 状态 | 操作 |
|---|---|---|---|---|---|
| 张三 | 100023 | 东港支行 | 机构管理员 | 启用 | 移除 |
| 李四 | 100041 | 莒县支行 | 机构管理员 | 启用 | 移除 |

顶部：

```text
[添加用户]
```

添加用户弹窗：

```text
搜索姓名 / 工号 / 机构

☑ 张三 100023 东港支行
☑ 李四 100041 莒县支行

[取消] [添加至角色]
```

规则：

- 一个用户允许绑定多个角色。
- 最终权限取角色权限并集。
- 数据范围建议取最宽范围，但页面需要明确提示。
- 角色调整后需重新登录或刷新权限。

## 10. 用户管理页面

### 页面结构

```text

查询区
姓名｜工号｜机构｜角色｜状态

[查询] [重置]

```

列表字段：

| 姓名 | 工号 | 登录账号 | 所属机构 | 角色 | 数据范围 | 状态 | 操作 |
|---|---|---|---|---|---|---|---|

操作：

```text
查看
分配角色
维护数据范围
启用/停用
```

规则：

- 用户基础信息来源于 SSO / 行内用户体系。
- 系统不新增行内用户。
- 系统不维护登录密码。
- 停用只影响本系统访问，不影响行内账号。

## 11. 机构管理页面

页面用于维护：

- 机构编码
- 机构名称
- 机构简称
- 机构类型
- 上级机构
- 机构层级
- 机构状态

建议布局：

```text
左侧：机构树
右侧：机构详情
```

示例：

```text
日照银行
├── 总行
│   ├── 消保岗
│   └── 总行部室
├── 青岛分行
├── 莒县支行
└── 东港支行
```

操作：

- 查看详情
- 编辑
- 启用/停用
- 维护上级机构

规则：

- 机构用于派单、数据范围、台账和报表统计。
- 机构停用不能影响历史工单归属。
- 机构变更需要留痕。

## 14. 给 AI 的页面设计指令

请在现有“日照银行智慧消保平台”中新增一个“后台管理”分支，不要设计成独立后台系统。保留当前系统原有的左侧导航、顶部栏、消息提醒和个人信息区域，仅在右侧主内容区新增后台管理页面。

左侧主导航增加“后台管理”，其下包含“用户管理、角色权限管理、机构管理、短信模板维护”四个子菜单，仅超级管理员可见。

首先设计“角色权限管理”页面。页面采用左右双栏结构：左侧为角色列表，右侧为角色详情和权限配置。

左侧角色列表展示“机构管理员、投诉处理员、超级管理员”，每张卡片显示角色名称、关联用户数、数据范围和启用状态。

右侧顶部展示角色基本信息，下方使用“菜单权限、按钮权限、数据范围、关联用户”四个 Tab。

菜单权限采用树形勾选；按钮权限按工单录入、工单处理、台账、报表等模块分组；数据范围采用“仅本人、本机构、本机构及下辖机构、指定机构、全行”单选；关联用户使用表格展示并支持添加用户。

页面整体严格沿用现有日照银行 UI 规范：橙色作为主强调色，白色卡片，浅灰背景，16px 圆角，24px 页面边距，主要按钮使用橙色，次级按钮使用白底描边。

不要设计复杂低代码权限平台，不要使用独立后台模板，不要改变现有系统框架。

## 15. 最终推荐

后台管理作为现有系统中的一个分支，结构建议：

```text
后台管理
├── 用户管理
├── 角色权限管理（核心）
├── 机构管理
└── 短信模板维护
```

最先完成的页面流程：

```text
角色列表
    ↓
选择角色
    ↓
配置菜单权限
    ↓
配置按钮权限
    ↓
配置数据范围
    ↓
关联用户
    ↓
保存生效
```

这套结构既能满足现有三类角色，也方便后续增加新角色，不需要重新开发页面。
